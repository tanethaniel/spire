## Reviewer Concerns (non-blocking)
- Privacy: No policy direction stated for voice recordings of emotionally vulnerable content. Recommend at minimum: all data encrypted at rest, user can delete all data anytime, no voice audio retained after transcription.
- Tester recruitment: "20 test users identified before building" has no fallback. If the founder can't recruit 20, start with 10 or even 5 -- the qualitative signal matters more than the sample size at this stage.
Mode: Startup

## Problem Statement

75% of people who start journaling quit. The three barriers are friction (typing feels like work), blank-page paralysis (not knowing what to write), and delayed gratification (benefits of journaling take weeks to feel). Spire bets these are product design failures, not motivation failures. Voice input lowers friction, calendar context eliminates the blank page, and data analytics deliver immediate reward through pattern recognition.

The target user is Gen-Z, wellness-oriented professionals starting their careers who want to understand themselves better. They know they should journal. They don't. Their current solution is nothing -- they're not journaling badly, they're not journaling at all. The competitor is inertia.

## Demand Evidence

Thesis-level, not behavior-level. No specific user has asked for this product. No prototype exists. No one would be upset if it disappeared because it doesn't exist yet. The founder cites psychological research (75% journaling dropout rate) and identifies three structural barriers, but has not validated that solving these barriers changes behavior.

This is a conviction bet. The strongest analog is Duolingo (created a learning habit where none existed) and Headspace (created a meditation habit where none existed). Both succeeded by making the first session magical and the daily commitment tiny.

## Status Quo

The target user processes their day by... not processing it. They scroll Instagram at 10pm and go to bed feeling vaguely unfulfilled. Some talk to friends. Some have tried journaling apps (Day One, Notion, paper) and abandoned them within weeks.

The status quo is doing nothing. This is the hardest competitor to beat because there's no active pain to replace -- only a latent belief that journaling would be good for them.

## Target User and Narrowest Wedge

Target: Gen-Z professionals (22-30), wellness and self-improvement oriented, starting their careers, wanting to understand themselves better. No specific individual named during the diagnostic.

Narrowest wedge: A single voice journal entry flow. User sees today's calendar events, taps one, gets a contextual prompt, talks for 30-120 seconds, sees transcript + themes + one AI follow-up question. That's it. If this single experience is magical enough to make someone come back tomorrow, everything else can be built on top.

## Constraints

- Pre-product: no code, no users, no revenue
- Solo founder (assumed)
- Must test core thesis before investing in full product
- The first session must convert: time from app open to completed entry under 90 seconds, user sees personalized insight within 5 seconds of finishing recording, zero typing required for core flow. Day-2 unprompted return is the primary validation metric.

## Premises

1. Voice input meaningfully lowers the friction barrier compared to typing -- enough to convert non-journalers into journalers.
2. Calendar context ("You had a 1:1 with your manager at 2pm") creates better prompts than generic prompts ("How was your day?").
3. Data analytics and pattern recognition provide enough "immediate reward" to replace the delayed gratification problem.
4. The primary competition is inertia (doing nothing), not existing journaling apps.
5. (REVISED) The first session's interface quality wins the second session. Light gamification bridges the first week. By week 2, accumulated analytics become the primary retention driver -- users come back to see what Spire has learned about them. Hard streaks risk creating guilt-based off-ramps in a private, emotionally vulnerable activity.

## Cross-Model Perspective

An independent Claude subagent reviewed the diagnostic cold. Key findings:

**Steelman:** "Spire is betting that the 75% journaling dropout rate is a product design failure, not a motivation failure. The wedge is that no one has attacked all three barriers in a single product aimed at the specific moment someone is already primed for reflection."

**48-hour prototype:** Voice-memo-to-insight loop. Calendar events, contextual prompt, 30-120 second voice entry, transcript + 3 themes + one follow-up question. Skip everything else. Ship to 20 people. Measure unprompted day-2 return.

## Approaches Considered

### Approach A: Voice-First MVP (CHOSEN)
One screen. Calendar events listed. Tap one, get contextual prompt. Hold button and talk for 30-120 seconds. See transcript + 3 extracted themes + one AI-generated follow-up question. Optionally respond to follow-up. Done. Entry saved.

Ship to 20 friends/contacts. Primary metric: unprompted day-2 return rate. Above 40% = something real. Below 20% = thesis needs revision. Note: with n=20, 40% is 8 people -- small-sample noise is real. Supplement with qualitative signal: text non-returners "did you try it again?" If 5+ say they wanted to but forgot, that's desire without habit formation (a fixable problem, not a thesis failure).

Effort: S (2-3 weekends). Risk: Low.

Tech stack: Mobile-first PWA (fastest to ship and share via URL -- no App Store review, no TestFlight). Whisper API (OpenAI) for speech-to-text, Claude API for contextual prompts + theme extraction + follow-up questions, Google Calendar API for event context, Supabase for auth + storage.

Theme output examples (for a sample entry "My 1:1 with Sarah went well, she liked my proposal but I'm nervous about the deadline"):
- Career confidence
- Time pressure
- Relationship with manager

Calendar fallback: When calendar events have low-information titles ("Meeting", "Busy", "Block"), fall back to time-of-day heuristics ("It's the end of your workday -- what's on your mind?") or let the user free-pick from a short prompt menu ("Work", "Relationships", "Something on my mind"). The calendar is the preferred prompt source but not a hard dependency.

### Approach B: Full Loop MVP
Voice + calendar + analytics dashboard + light gamification. 4-6 week build. Tests the full thesis including the retention handoff, but risks building features nobody wants before validating the core experience.

### Approach C: AI Conversation Partner
Skip journaling entirely. Build an AI that has a 3-minute reflective conversation at end of day using calendar context. Most novel direction, but highest technical risk (real-time voice AI) and trust concerns (AI + vulnerability).

## Recommended Approach

Approach A: Voice-First MVP. It answers the only question that matters right now: does the first voice journal entry feel magical enough that someone comes back tomorrow?

If the answer is yes (day-2 return >40%), build Approach B (analytics, gamification, polish). If the answer is no, you've spent a weekend, not 6 weeks, learning that voice + calendar alone isn't enough.

## Open Questions

1. Will Gen-Z users talk into their phone for reflection, or does voice feel awkward for vulnerable content?
2. How contextual can calendar prompts be? "You had a meeting with Sarah" is useful. "You had an event called 'Meeting'" is not.
3. What does the analytics retention handoff actually look like? What insight at week 2 is compelling enough to keep someone journaling?
4. Is the right form factor a native mobile app (habit-forming, notifications) or a web app (faster to build, lower distribution friction)?
5. Privacy: where is voice data stored? Transcripts? This is deeply personal content.

## Success Criteria

- Day-2 unprompted return rate >40% among first 20 users
- At least 3 users complete 7 consecutive days of entries
- At least 1 user describes the experience as "magical," "surprisingly easy," or equivalent
- Total build time for MVP: under 3 weekends
- Time from app open to completed entry: under 90 seconds
- Zero typing required for core journal flow

## Distribution Plan

Phase 1 (MVP test): Direct share via URL (PWA -- no App Store, no TestFlight needed). Send to 20 personal contacts. No public distribution.

Phase 2 (if validated): App Store (iOS first -- the wellness demographic is iPhone-heavy). Consider ProductHunt launch for initial traction.

CI/CD: Deferred. Manual deploy (Vercel or similar) for Phase 1. Set up GitHub Actions for Phase 2. Move to native (Expo/React Native) only if retention validates and notifications become the bottleneck.

## Dependencies

- Google Calendar API access (OAuth consent screen setup)
- Speech-to-text API account (Whisper API via OpenAI or Deepgram)
- Claude API account for prompt generation and theme extraction
- 20 willing test users identified before building

Ask them: "Why did you stop?" Write down their exact words. If none of them say anything related to friction, blank pages, or delayed gratification, your thesis needs revision. ## Engineering Architecture (from /plan-eng-review 2026-06-01)

### Finalized Stack

```
BROWSER (PWA, mobile-first — React + Vite → Vercel)
    │
    ├── Optional: camera/upload → calendar screenshot
    ├── MediaRecorder API → audio (codec detection: webm on Chrome, mp4 on Safari)
    ├── IndexedDB (idb) → audio buffer per question (deleted after transcript confirmed)
    │
    └── Supabase Edge Functions (API proxy — keys never in browser)
            ├── POST /extract-events  → Claude Vision → event list from screenshot
            ├── POST /generate-q1     → ElevenLabs TTS (dynamic, uses calendar context)
            └── POST /process-entry   → Whisper (transcribe) + Claude (themes)

SUPABASE
    ├── Auth: email magic link
    ├── DB: journal_entries (RLS enabled — users can only read own entries)
    │       q1_transcript, q2_transcript, ..., q6_transcript, themes[], event_context
    └── DB: user_events (session_open + source tag, question_started/completed per index)

STATIC ASSETS (Vercel)
    └── public/audio/q2.mp3 ... q6.mp3 (ElevenLabs pre-generated, instant playback)
```

### 6-Question Guided Session

ElevenLabs speaks each question. User holds button to record their answer (~1 min each). Background transcription: Q1 uploads to Whisper while ElevenLabs reads Q2. Total target: under 7 minutes.

```
Q1: [Context-aware, dynamic TTS]
    - With calendar screenshot: "You had a meeting with Sarah today — what stood out?"
    - Without: "What did you do today?"
Q2: How did those things make you feel? What emotions stemmed from today?
Q3: What memories did you make today? What stuck with you?
Q4: Was there anything interesting you learned today?
Q5: Was there anything interesting you learned about yourself today? What caused it?
Q6: Anything else?
```

After Q6: Claude analyzes all 6 transcripts → extracts 3 themes → saves entry. Progressive reveal: transcript appears → themes surface.

### Session State Machine

```
IDLE
  └─► CALENDAR_UPLOAD (optional)
        └─► GENERATING_Q1 (ElevenLabs, ~1-3s)
              └─► [ROUND 1-6]
                    TTS_PLAYING → RECORDING → BACKGROUND_TRANSCRIBING → next TTS_PLAYING
              └─► ANALYZING (Claude — all 6 transcripts)
                    └─► RESULT | ERROR
```

### Error Handling

| Stage | Failure | User experience |
|-------|---------|----------------|
| Whisper | Timeout / 5xx | Retry button. Audio in IndexedDB — user doesn't re-record. |
| ElevenLabs | Any error | Silent fallback: question shown as text. User taps to start recording. |
| Claude | Timeout | Transcripts saved without themes. User sees raw entries. |
| Mic permission | Denied | Clear instructions to enable in Settings + restart session. |

### Key Architectural Decisions

- **API keys**: Supabase Edge Functions only. Never in browser bundle.
- **RLS**: Enabled on `journal_entries` from migration 001. Users own their rows.
- **iOS Safari**: Runtime codec detection (`audio/mp4` fallback). Audio session must start in `touchstart` handler — no `setTimeout` delay.
- **Q1 TTS**: Dynamic (ElevenLabs at session start with calendar context). Q2-Q6: pre-generated static files.
- **Audio privacy**: Edge Function streams audio to Whisper, discards immediately. No audio stored.
- **Metric integrity**: No engagement emails for 48h after first session. `session_open.source` tag distinguishes direct vs. email-driven return.

### NOT in Scope (V1)

- Google Calendar OAuth (deferred to Sprint 2 — use screenshot + Claude Vision for now)
- Analytics dashboard
- Gamification / streaks
- Native mobile app (Expo/React Native)
- CI/CD (manual Vercel deploy for Phase 1)
- AI-generated follow-up questions (standardized 6 questions in V1)

### Distribution

Phase 1: Manual Vercel deploy → share URL directly with 20 contacts. No App Store.
Phase 2 (if validated): iOS App Store first (wellness demographic is iPhone-heavy). Consider ProductHunt.

## UI Design Specifications (from /plan-design-review 2026-06-01)

### Three Screens

**Screen 1 — Home / Prompt Selection**
- Header: `spire.` wordmark (left) + avatar initial (right)
- Greeting line: time-of-day contextual ("Monday evening") + "What's been on your mind today?"
- Calendar zone: dashed border card with camera icon + "Add today's calendar" title + "Screenshot → better prompts" sub. Tap → bottom sheet: "Choose from library" | "Take screenshot". Loading: spinner. Success: "Found 4 events" → events replace the upload zone as context chips.
- Skip link: "Skip — reflect without calendar" centered below zone
- Divider: thin line with "or choose a topic" label
- Topic grid: 2×2, pills with emoji icon + label + hint text. Options: Work (💼), People (🤝), Mind & body (🌿), On my mind (💭)
- Selected prompt preview: appears below grid with amber left-border, shows exact Q1 that will be asked
- CTA: full-width amber button "▶ Start reflecting" with sub-copy "6 questions · ~7 minutes · voice only"
- Day-2+ returning user state: subtle chip at top of screen showing yesterday's top theme ("Yesterday: Career confidence") — creates analytics pull

**Screen 2 — Guided Session (6 Questions)**
- Progress: back button (left) + "Question N of 6" label (right) + thin progress bar + 6 dots (pill-shaped active dot, filled past dots, empty future dots)
- Speaking state: small badge "Spire is speaking…" with animated waveform. Record button is LOCKED (grayed, no interaction) while ElevenLabs audio plays. Badge fades + button activates when audio ends.
- Question number: `Q1` in amber uppercase label
- Question text: 26px semibold, calendar-injected name in amber (`<em>` styling)
- Sub-prompt: 15px muted, "Take your time. There's no wrong answer."
- Live transcript area: dark inset card, text appears as user speaks (via real-time display of Whisper incremental results — or simulated in V1). Shows "Your words will appear here…" placeholder when idle.
- Record button states:
  - Locked (ElevenLabs playing): gray fill, slow pulse
  - Ready: amber fill, "Hold the button to answer" hint
  - Recording: dark background + red border + stop square icon + ripple rings radiating
  - Done: "✓ Saved — transcribing in background" status
- Recording < 5 seconds: on release, toast appears — "That was quick — want to re-record?" with Yes/Skip
- ElevenLabs fallback (error): badge hides, question shown as large text, record button activates immediately
- Skip: small "Skip this question" text button below record area (min 44px touch target)
- Network lost: toast "Lost connection — your recording is saved locally"
- Crash recovery: on app reopen, modal — "You were mid-session — resume?" with Yes/Start over

**Screen 3 — Progressive Result Reveal**
- Loading state: centered spinner + "Reading your reflection…" (1.8s before reveal)
- Progressive reveal sequence:
  1. Transcripts accordion appears (all collapsed by default)
  2. Theme chips animate in staggered (0.2s apart): each chip has a colored dot + specific label
  3. AI insight card fades in (gradient top border, amber label "✦ Spire noticed", italic insight text quoting user's words, two action buttons: "Save for later" | "Reply with voice")
  4. "Done for today ✓" CTA + day streak note appears
- Accordion transcripts: `Q1 · Context` label in amber, first 60 chars of transcript as preview, expand to full
- Error state (Claude themes timeout): transcripts still appear, theme area shows 3 skeleton chips with muted "Themes unavailable" label
- "Done for today ✓" → brief success animation → navigate to home state showing "See you tomorrow" + today's themes

### Accessibility Requirements

- Record button: `aria-label="Hold to record"`, `aria-pressed` state, min 80px (already 80px ✓)
- Question text: wrapped in `aria-live="polite"` region — screen reader announces new question
- Skip links: min 44px touch target height (currently 30px — FIX REQUIRED)
- ElevenLabs audio: visual speaking indicator must be present for deaf/hard-of-hearing users who need to know when speech ends — the waveform badge serves this role
- Color contrast: `--text-ghost` raised to `#5A5958` (3.2:1 on base) for all functional text. Purely decorative dividers may use lower contrast.
- Viewport minimum: 375px (iPhone SE). Prompt grid must not overflow on 375px.

### Claude Prompt Constraints (Design-Critical)

**Theme extraction:** Themes must be 2–4 word specific noun phrases derived from user's exact language. Forbidden: single-word categories ("Work"), emotional adjectives ("Positive feelings"), universal labels ("Stress"). Valid examples: "Career confidence," "Manager relationship," "Q3 deadline pressure."

**Insight card copy:** Must quote or closely paraphrase user's actual words. Must ask a question that couldn't apply to anyone else. Forbidden openers: "It sounds like you care about...", "You seem to value...", "Today was clearly...". Insight must be grounded in specific language from the session.

### Unresolved (post-V1)

- Auto-advance timing after each answer: 2s default, but Q6 ("Anything else?") should have longer silence before advancing
- Whether to show "Share entry" on result screen
- Whether background transcription (Q1 uploading while Q2 plays) surfaces any loading state to user
